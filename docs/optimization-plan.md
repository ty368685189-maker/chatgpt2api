## 任务卡

### T1 图片回归测试加固

### T2 注册流程校验

目标：

- 在前端尽量拦住明显无效配置，减少启动后才失败的情况。

任务：

- [x] CloudMail 域名必填校验
- [x] CloudMail 域名提示和保存前说明统一
- [x] 注册页补更多 provider 的必填项提示
- [x] 注册页统一错误文案和后端校验结果
- [x] 让保存和启动前的校验逻辑保持一致

完成标准：

- 用户在保存和启动时看到的校验规则一致。
- 配置明显不完整时，页面先报错而不是后端再失败。
